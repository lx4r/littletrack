# Git & Documentation

## Commit Messages

- Focus on the "why" (intent/motivation) and the behavioral "what" (not code-level description)
- Keep the subject line concise; use the body for detail when needed

## Documentation

- Update README and potentially `AGENTS.md` when adding/changing user-facing behavior
- Keep README examples working and representative

## Maintaining `AGENTS.md`

- Alert the user if work appears to violate these guidelines
- Suggest updates when new patterns or conventions emerge that should be documented
